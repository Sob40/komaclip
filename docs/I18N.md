# Internationalization

KomaClip is English-first with Spanish selectable.

## Locale Sources

The interface locale is resolved in this order:

1. Explicit route locale, such as `/es`.
2. Signed-in user's `users.locale`.
3. Rails default locale, `en`.

Project content language is a separate future concern. A user can browse the UI
in Spanish while creating English marketing clips, or the reverse.

## Routing

English uses unprefixed URLs by default:

```text
/
/session/new
/app
```

Spanish uses the `/es` prefix:

```text
/es
/es/session/new
/es/app
```

Because the locale segment appears before resource identifiers, token-based
helpers should use named arguments:

```erb
<%= edit_password_url(token: @user.password_reset_token) %>
```

This avoids confusing a token for the optional locale segment.

## Catalogs

Imported catalog labels currently use English source text. Spanish catalog
labels should be added through product-owned localization, not inside renderer
code. The UI locale and catalog/content locale must remain separate.
